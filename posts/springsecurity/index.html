Spring Security 개요

Spring Security 는 Servlet Filter를 이용하여 Service의 보안과 관련된 기능을 한곳에서 모아 처리하는 기능을 제공합니다.

Filter 는 Java Servlet Specification에 정의된 HttpServletRequest가 Servlet까지 오기 전에 사전(후) 처리하는 방법입니다.

Spring Security와 관련된 동작들은 기본적으로 이 Filter layer에서 모두 이루어집니다. 참고로 아래 그림은 Spring Security Architecture 의 그림을 보강한 것입니다.

위 Spring Security 는 위와 같은 FilterChain은 아래와 같은 Java Configuration 형태로 만들 수 있도록 합니다. WebSecurityConfigurerAdapter 당 FilterChain 이 하나씩 만들어집니다.

@EnableWebSecurity  // 동작을 관찰하려면 (debug=true)
public class WebSecurityConfig{

    @Configuration
    protected void configure(HttpSecurity http) throws Exception{
        http.antMatcher("/api/**")  //filterchain
              .authorizeRequests()  //authorization
              .antMatchers("/api/filters").permitAll()
              .antMatchers("/api/url2/**").hasRole("ADMIN")
              .antMatchers("/api/url3/**").access("hasRole('ADMIN') and hasRole('DBA')")
              .anyRequest().denyAll()
            .and()                 //authentication
              .httpBasic()
            .and()
            .csrf().disable()      //csrf
            .sessionManagement().disable();  //session
    }

    @Order(1)
    @Configuration
    public static class Config2 extends WebSecurityConfigurerAdapter {
        protected void configure(HttpSecurity http) throws Exception{
            http.antMatcher("/internal/**").httpBasic().and()
            .authorizeRequests().antMatchers("/test/**").authenticated();
        }
    }
}

Spring Security 를 이용하여 어플리케이션을 만들기 위해서는 아래와 같은 것들을 설정 및 개발하여야 합니다.

  • 만들고자 하는 서비스의 보안 모델을 설정
  • Spring Security Filter Chain 를 설정
  • Authentication 관련 설정, 개발
  • Authorization 관련 설정, 개발
