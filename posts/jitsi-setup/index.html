Jitsi Single Instance Setup How to

 Posted on April 19, 2020  |  3 minutes  |  488 words  |  Yong-in SCV

Ref

  • DigitalOcean’s Jitsi Install guide
  • Jitsi Official Install guide

Steps

  • 위 Digital Ocean’s Guide에 Jitsi Official guide의 NAT관련 설정을 더 해주면 됨.
  • Jitsi 는 기본 설치하면 “인증”이 아무것도 없음. 누구나 들어와서 화상 회의를 개설 할 수 있음. 그런데 위 Digital Ocean’s guide는 기존 인증을 세팅하도록 가이드하고 있어 좋음
  • Linux Mem 1G 이상은 필요한듯 (512MB 사용 시 video bridge OOM남)
  • public domain 이 있어야 편함. https 인증서는 jitji script 이용하면 let’s encrypt 로 부터 받음. 아래 가이드에서는 \$domain 으로 표시함

절차

### hostname 설정
sudo hostnamectl set-hostname $domain

# /etc/hosts 에도 $domain 추가
127.0.0.1 localhost $domain

### 사용하는 포트: 80, 443, 4443, 1000/udp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 4443/tcp
sudo ufw allow 10000/udp
sudo ufw status
# Amazon 이면 Security group에도 추가

### install jitsi
echo 'deb https://download.jitsi.org stable/' >> /etc/apt/sources.list.d/jitsi-stable.list
wget -qO - https://download.jitsi.org/jitsi-key.gpg.key | sudo apt-key add -
apt-get install apt-transport-https
apt-get update
apt-get -y install jitsi-meet
# 설치 중간에 domain 물어보면 $domain
# 인증서 물어보면 generate a new self-signed .. 선택

### lets encrypt 인증서 설치
/usr/share/jitsi-meet/scripts/install-letsencrypt-cert.sh

### NAT관련 처리(ec2 instance 에서도 필요)
# /etc/jitsi/videobridge/sip-communicator.properties 에서 아래 추가
# 이거 설정 안하면 두명까지는 비디오콜 잘 되는데 3명부터 안됨
org.ice4j.ice.harvest.NAT_HARVESTER_LOCAL_ADDRESS=<Local.IP.Address>
org.ice4j.ice.harvest.NAT_HARVESTER_PUBLIC_ADDRESS=<Public.IP.Address>

### 기본 인증 설정(이 단계를 거치지 않으면 누구나 화상회의 개설 가능)
# vi /etc/prosody/conf.avail/$domain.cfg.lua
# 에서 authentication = "anonymous" 로 되어 있는 줄을 찾아서 internal_plain 으로 변경
        authentication = "internal_plain"
# 그리고 해당 파일 제일 밑에 아래 추가
VirtualHost "guest.$domain"
    authentication = "anonymous"
    c2s_require_encryption = false

# vi /etc/jitsi/meet/your_domain-config.js
# 에서 // anonymousdomain: 'guest.example.com', 으로 되어 있는것을 수정
        anonymousdomain: 'guest.$domain',

# /etc/jitsi/jicofo/sip-communicator.properties 에서도 아래 추가
org.jitsi.jicofo.auth.URL=XMPP:$domain

# 화상회의 개설할 수 있는 어드민용 username, password 추가
# sudo prosodyctl register user your_domain password
sudo prosodyctl register admin $domain adminpassword

### (Optional) 화질 조정
# /etc/jitsi/meet/$domain-config.js 에서 아래 관련 부분 수정
# defualt 는 720p라 network bandwidth 걱정되면 조정.
resolution: 480,
constraints: {
      video: {
          aspectRatio: 16 / 9,
          height: {
              ideal: 480,
              max: 480,
              min: 180
          }
      }
},

### Jitsi server restart
sudo systemctl restart prosody.service
sudo systemctl restart jicofo.service
sudo systemctl restart jitsi-videobridge2.service
